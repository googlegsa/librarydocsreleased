Configuration values for this program, like the GSA's hostname. Also several knobs, or controls, for changing the behavior of the program.

All available configuration:

required? namemeaning
adaptor.autoUnzip expand zip files and send each file inside separatly. Defaults to false
adaptor.sendDocControlsHeader use X-Gsa-Doc-Controls HTTP header with namespaced ACLs. Otherwise ACLs are sent without namespace and as metadata. Defaults to false
adaptor.fullListingSchedule when to invoke Adaptor.getDocIds, in cron format (minute, hour, day of month, month, day of week). Defaults to 0 3 * * *
adaptor.incrementalPollPeriodSecs number of seconds between invocations of PollingIncrementalAdaptor.getModifiedDocIds. Defaults to 900
adaptor.docContentTimeoutSecs number of seconds before a call to get document content times out and is interrupted. Defaults to 30
adaptor.pushDocIdsOnStartup whether to invoke Adaptor.getDocIds on process start (in addition to adaptor.fullListingSchedule). Defaults to true
docId.isUrl say your adaptor's document ids are already URLs and avoid them being inserted into adaptor generated URLs. Defaults to false
feed.crawlImmediatelyBitEnabled send bit telling GSA to crawl immediately. Defaults to false
feed.maxUrls set max number of URLs included per feed file. Defaults to 5000

feed.noRecrawlBitEnabled send bit telling GSA to crawl your documents only once. Defaults to false
gsa.614FeedWorkaroundEnabled enable detour around particular feed parsing failure found in GSA version 6.14 . Defaults to false

getServerFullAccessHosts

public String[] getServerFullAccessHosts()
Comma-separated list of IPs or hostnames that can retrieve content without authentication checks. The GSA's hostname is implicitly in this list.

When in secure mode, clients are requested to provide a client certificate. If the provided client certificate is valid and the Common Name (CN) of the Subject is in this list (case-insensitively), then it is given access.

In non-secure mode, the hostnames in this list are resolved to IPs at startup and when a request is made from one of those IPs the client is given access.

getServerBaseUri

public URI getServerBaseUri()
Optional: Returns this host's base URI which other paths will be resolved against. It is used to construct URIs to provide to the GSA for it to contact this server for various services. For documents (which is probably what you care about), the getServerBaseUri(DocId) version is used instead.

It must contain the protocol, hostname, and port, but may optionally contain a path like /yourfavoritepath. By default, the protocol, hostname, and port are retrieved automatically and no path is set.

getServerBaseUri

public URI getServerBaseUri(DocId docId)
Optional: Returns the host's base URI which GSA will contact for document information, including document contents. By default it returns getServerBaseUri(). However, if you would like to direct GSA's queries for contents to go to other computers/binaries then you can change this method.

For example, imagine that you want five binaries to serve the contents of files to the GSA. In this case you could split the document ids into five categories using something like: 

String urlBeginnings[] = new String[] {
   "http://content-server-A:5678",
   "http://content-server-B:5678",
   "http://backup-server-A:5678",
   "http://backup-server-B:5678",
   "http://new-server:7878"
 };
 int shard = docId.getUniqueId().hashCode() % 5;
 return URI.create(urlBeginnings[shard]);

Note that this URI is used in conjunction with getServerDocIdPath() and the document ID to form the full URL. In addition, by using getServerBaseUri() and getDocIdPath(), we have to be able to parse back the original document ID when a request comes to this server.

isServerSecure

public boolean isServerSecure()
Whether full security should be enabled. When true, the adaptor is locked down using HTTPS, checks certificates, and generally behaves in a fully-secure manner. When false (default), the adaptor serves content over HTTP and is unable to authenticate users (all users are treated as anonymous).

The need for this setting is because when enabled, security requires a reasonable amount of configuration and know-how. To provide easy out-of-the-box execution, this is disabled by default.
